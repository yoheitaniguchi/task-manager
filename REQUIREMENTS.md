# Obsidianタスク管理プラグイン 要求定義書 v2（Opus 4.8 引き継ぎ用）

> **Opus 4.8への指示：** 本要求定義に基づき、次の順で進めてください。
> ①TypeScriptでのプラグイン詳細設計（クラス構成・ファイル構成・データフロー図）
> ②MVP実装（後述の実装順序に従う）
> 要求定義は確定済みです。未決事項はありません。設計上の技術判断が必要な場合のみ質問してください。

---

## 1. 背景と目的

- 利用者：システム開発チームの係長（6名チーム管理、会議多数、ベンダーマネジメント併任）
- 課題：(1) マルチタスク化による作業優先順位の曖昧化と効率低下、(2) 会議過多、(3) メンバーへのタスク委譲の追跡困難
- 方針：**パーソナルカンバン+WIP制御（リーン/TOC系）を骨格とし、会議の重要度×コスト評価（タイムブロッキング系）を統合したハイブリッド**

## 2. 設計原則

1. 本プラグインは**個人のフロー制御レイヤー**であり、記録システム（Backlog/Jira等）を代替しない
2. WIPは「強制」ではなく**「摩擦設計」**：破れるが、破ったことが可視化・記録される
3. すべてを構造化しない：**ボードに載せる価値があるものだけ昇格**する
4. 割り込みは禁止せず**計測**する（Expediteレーン）
5. 委譲は無料ではない：フォローアップコストをキャパシティに計上する

## 3. ドメインモデル

### 3.1 エンティティ

Task 1種類に統一。1タスク=1ノート、属性はfrontmatter。

```yaml
# 共通
type: task | meeting | delegated
status: backlog | ready | doing | waiting | done
sort_key: 小数（fractional indexing。並べ替え時の書き換えは移動ファイル1件のみ）
estimate: 見積時間（分）
due: 期限（任意）
project: プロジェクト名（任意）
ticket_url: チケットURL（任意。内容は複製しない。タイトル+URL+見積のみの軽量運用）
expedite: true | false（割り込みフラグ）
expedite_started: 割り込み着手日時（自動記録）
expedite_interrupted: 中断したタスクへのリンク（自動記録）

# type: meeting
meeting_role: decision | contribute | listen
replaceable: true | false
attendees: 参加人数
meeting_start: 開始日時
meeting_duration: 分

# type: delegated
assignee: 担当メンバー名
delegated_at: 依頼日
expected_by: 期待返答日
```

### 3.2 状態遷移

`backlog → ready → doing → done`。委譲・外部待ちは `waiting`（WIPカウント外）。
Expediteタスクは専用レーン（WIPカウント外、ボード最上段）。

## 4. 機能要求

### F1. カンバンボードビュー

- 列：Expedite（最上段横断レーン）/ backlog / ready / doing / waiting / done
- Dataview的クエリでVault内タスクノートを収集し自前描画（既存Kanban/Tasksプラグインに非依存。frontmatter互換性は維持）
- デスクトップ：ドラッグ&ドロップで状態変更。**モバイル：閲覧+タップメニューでの状態変更のみ（D&D非対応）**

### F2. WIP摩擦設計

- doing列のWIP上限：デフォルト3（設定変更可）
- UI上のD&Dは超過時ブロック
- frontmatter直接編集による超過はブロックせず：ボードヘッダー赤色表示+超過理由入力モーダル
- 超過発生をログ記録し、週次サマリーで回数集計

### F3. Expediteレーン（割り込み管理）

- `expedite: true` でWIP枠外の最上段レーンへ
- 着手日時・中断タスクを自動記録
- 週次でExpedite件数・消費時間を集計表示

### F4. 会議管理（ICS購読連携）

- 設定画面にICS購読URL（Google/Outlookの公開ICSリンク）を登録。定期取得+手動更新ボタン
- 当日の会議を type: meeting のタスクとして自動反映（既存ノートとの重複はUID等で排除）
- **会議評価：2軸4象限**
  - 重要度 = 役割係数（decision:3 / contribute:2 / listen:1）
  - コスト = 会議時間（分）× 参加人数 ÷ 60（人時）
  - **辞退・委任候補**：役割 listen かつ replaceable: true のみ（グレー表示+バッジ）
  - **短縮・少人数化の交渉候補**：高重要×高コスト（別ラベル表示）
  - replaceable: false は重要度に関わらず辞退候補から除外
- meeting_role / replaceable はICS取得後にユーザーがカード上で設定（未設定は contribute / false をデフォルト）

### F5. 日次キャパシティバー

- 日次キャパシティ = 可処分時間（設定値、デフォルト6h）− 当日会議時間合計 − （waiting件数 × フォローアップコスト[デフォルト10分/件、設定可]）
- doing列のestimate合計がキャパシティ超過で警告表示

### F6. デイリープランニングモード（朝の並べ直し儀式）

- 起動後初回表示時（または設定時刻以降の初回）に起動を提案
- ready列上位タスクをカード比較形式で提示し、当日キャパシティに収まる分だけ「今日の枠」に確定させるフロー
- 実施/スキップをログ記録（成功指標の計測用）

### F7. キャプチャ二層構造（昇格モデル）

- 第一層：デイリーノート内の素のチェックボックス（プラグイン非関与）
- 第二層：タスクノート
- チェックボックス上のコマンド/コンテキストメニューで**昇格（promote）**：タスクノートを生成し、元のチェックボックスをリンクに置換
- タスクノート直接生成コマンドも提供

### F8. 委譲リマインダー + フォローアップコスト計上

- assignee設定+waiting移動で委譲完了
- expected_by超過で警告。**委譲後7日間状態変化なしでも警告**
- Markdownエクスポート：依頼内容・期待返答日入りの定型文を生成（Slack等へのコピペ想定）
- 片方向管理（Vault共有・リアルタイム同期はスコープ外）。本機能の価値は「リマインダー+コスト可視化」と定義する

### F9. 腐敗検知（週次レビュー最小実装）

- backlog滞留14日超のカードにバッジ表示
- 週の初回起動時に3行サマリーをモーダル表示：滞留カードN件 / Expedite N回 / WIP超過N回

### F10. 設定画面

WIP上限 / 可処分時間 / フォローアップコスト / 役割係数 / ICS購読URL / プランニング提案時刻 / 滞留日数しきい値

## 5. 非機能要求・技術方針

- Obsidian Plugin API + TypeScript
- ファイル書き込み：楽観的更新+ファイル変更イベント購読で外部変更を即時再描画。書き込み前にmtime比較、競合検知時は上書きせず再読み込み（Obsidian Sync/他デバイス編集との共存）
- 並び順：fractional indexing（sort_key小数）
- モバイル：閲覧+タップ操作のみ対応
- ログ：WIP超過・Expedite・会議判断（辞退/短縮）・プランニング実施をローカルJSON等に記録
- 拡張ポイント：チケットAPI連携の口だけアーキテクチャ上確保（実装しない）

## 6. スコープ外（将来）

チケットシステムAPI連携 / Calendar API（OAuth）連携 / Vault共有によるチーム同期 / リードタイム等フロー指標の可視化 / 週次レビューレポート自動生成 / モバイルD&D

## 7. MVP受入基準

1. ボード表示・全状態変更が正常動作（デスクトップD&D、モバイルタップ）
2. WIP超過時の摩擦動作（ブロック/赤表示/理由入力/ログ）が仕様通り
3. ICS購読から当日会議が自動反映され、キャパシティバーが正しく計算される
4. 昇格コマンドが動作し、元チェックボックスがリンク置換される
5. 腐敗検知3種（滞留バッジ/expected_by超過/7日無変化）が発火する

## 8. 成功指標（リリース後4週間の自己運用で判定）

- WIP超過回数が週次で減少傾向
- 辞退・短縮した会議時間の累計が月2時間以上
- expected_by超過の検知漏れゼロ
- 毎朝のプランニング実施率80%以上

## 9. 実装順序（推奨）

1. データモデル+タスクノート生成/昇格コマンド（F7）
2. ボードビュー+状態遷移+WIP摩擦（F1, F2）
3. sort_key並べ替え+競合検知書き込み
4. ICS購読+会議反映+2軸評価（F4）
5. キャパシティバー（F5）
6. Expedite・委譲・腐敗検知（F3, F8, F9）
7. デイリープランニングモード（F6）
8. 設定画面・ログ・モバイルタップ操作（F10ほか）
