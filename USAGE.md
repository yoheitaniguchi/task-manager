# Task Manager 操作説明

Obsidian上で動く、個人用のタスク・フロー制御プラグインの使い方。

開発者向けの情報は [README.md](README.md)、要件は [REQUIREMENTS.md](REQUIREMENTS.md) を参照。

---

## 1. これは何か

**着手中の仕事を絞り込むための道具**であって、タスクの記録簿ではありません。

- 何を今日やるかを決め、同時進行を物理的に減らす
- 会議に払っているコストを数字で見る
- 人に投げた仕事を落とさない

Backlog や Jira の代わりにはなりません。それらは記録システムとして残したまま、
「自分が今どれを持っているか」だけをここで管理します。

---

## 2. 準備

### 有効化

設定 → コミュニティプラグイン → **Task Manager** を有効化。

### ボードを開く

- 左のリボンから **カンバンのアイコン**
- または `Ctrl+P` → 「タスクボードを開く」

### 最初にやる設定

設定 → Task Manager で、少なくとも次の2つは自分の実態に合わせてください。

| 項目 | 既定値 | 目安 |
|---|---|---|
| WIP上限 | 3 | 同時に着手してよい件数。まず3で始めて、破りすぎるなら実態に寄せる |
| 1日の可処分時間 | 360分 | 会議を除いて実作業に充てられる時間。6hは楽観的なことが多い |

---

## 3. 1日の流れ

```
朝  ─→ デイリープランニングで「今日の枠」を決める
    ─→ 会議を取得してキャパシティを確認
日中 ─→ Ready から Doing へ引く（WIP上限まで）
    ─→ 割り込みが来たら Expedite にする
夕方 ─→ 終わったものを Done へ、待ちに入ったものを Waiting へ
```

---

## 4. 画面の見かた

### キャパシティバー（上部）

```
可処分 6h00m − 会議 2h30m − フォローアップ 0h30m = 使える 3h00m ／ doing 合計 4h00m
                                                              ⚠ キャパシティ超過 1h00m
```

- **使える時間** = 可処分時間 −（当日の会議時間）−（Waiting件数 × フォローアップコスト）
- Waiting が引かれるのは、**人に投げた仕事もこちらの時間を食う**という前提だからです
- Doing の見積合計が「使える時間」を超えるとバーが赤くなります

### Expediteレーン（最上段）

割り込み仕事の専用レーン。**WIP上限にカウントされません**。

ここへはドラッグでは入れられません。カードのメニューから「Expedite にする」を選ぶと移動します。
（所属を決めるのは位置ではなくフラグのため）

### 列

| 列 | 意味 |
|---|---|
| Backlog | いつかやる。滞留すると警告が出る |
| Ready | 今日〜近日中にやると決めたもの |
| Doing | 着手中。**WIP上限の対象** |
| Waiting | 人待ち・外部待ち。WIP上限の対象外だがキャパシティは消費する |
| Done | 完了 |

Doing列のヘッダーには `2 / 3` のように現在数と上限が出ます。超過すると赤くなります。

### カードのバッジ

| 表示 | 意味 |
|---|---|
| `滞留 15日` | Backlogに設定日数より長く放置されている |
| `返答期限 3日超過` | 委譲の `expected_by` を過ぎた |
| `7日 無変化` | 委譲後、状態が動いていない |
| `辞退・委任候補` | 出なくてよい会議（カードが灰色になる） |
| `短縮・少人数化の交渉候補` | 重要だが高コストな会議 |

## 5. 操作一覧

### デスクトップ

| したいこと | 操作 |
|---|---|
| 列を移動 / 並べ替え | カードを**ドラッグ&ドロップ** |
| メニューを出す | カードを**右クリック** |
| ノートを開く | カードを**クリック** |

### モバイル

ドラッグ&ドロップは使えません。**カードをタップするとメニュー**が出るので、そこから移動します。

### カードのメニュー

- 各列への移動
- Expedite にする / 解除
- （会議のみ）役割の変更、代替可能の切り替え
- （委譲のみ）委譲メッセージを作成
- ノートを開く

### コマンド（`Ctrl+P`）

| コマンド | 用途 |
|---|---|
| タスクボードを開く | ボード表示 |
| タスクノートを作成 | 空のタスクノートを作る |
| チェックボックスをタスクノートに昇格 | 後述 |
| デイリープランニングを開く | 今日の枠を決める |
| 会議を取得（ICS） | カレンダーから当日の会議を取り込む |

---

## 6. タスクの作り方

### 二層構造という考え方

**思いついたことを全部カードにしないでください。** ボードが埋まると機能しません。

```
第一層：デイリーノートの素のチェックボックス   ← プラグインは関与しない。ここに雑に書く
   ↓ 昇格（本当にやると決めたものだけ）
第二層：タスクノート                          ← ボードに載る
```

### 昇格のしかた

1. デイリーノート等で `- [ ] 見積書をレビューする` の行にカーソルを置く
2. `Ctrl+P` → 「チェックボックスをタスクノートに昇格」

タスクノートが作られ、元の行は `- [ ] [[見積書をレビューする]]` に置き換わります。
デイリーノート側の見た目は保たれるので、後から辿れます。

### 直接作る

`Ctrl+P` → 「タスクノートを作成」。作成先フォルダは設定で変更できます（既定 `Tasks`）。

### 委譲タスクを作る

**現在、専用のコマンドはありません。** タスクノートを作ってから、frontmatterを手で編集します。

```yaml
type: delegated
assignee: 田中
expected_by: 2026-08-01
```

その後 Waiting へ移すと、委譲した日付が自動で記録され、追跡が始まります。

## 7. WIP上限の挙動

同時着手を減らすための仕掛けです。**禁止ではなく摩擦**として設計しています。

| 破り方 | 起きること |
|---|---|
| ボード上でDoingに4件目をドラッグ | **ブロックされます**。通知が出て移動しません |
| frontmatterの `status` を直接編集して4件にする | ブロックできないので、**列ヘッダーが赤くなり、理由の入力を求められます** |

理由を入力すると記録され、週次サマリーで「WIP超過 N回」として集計されます。
入力を「あとで」にしても超過状態は解消されません。記録されないだけです。

**破れるようにしてあるのは意図的です。** 破れないルールは形骸化するので、
破った事実が残るようにしてあります。

---

## 8. 会議

### カレンダーを繋ぐ

1. Google/Outlook で**公開ICSリンク**を取得
2. 設定 → Task Manager → ICS購読URL に貼る（1行に1つ、複数可）
3. 「いま会議を取得する」またはボード上部の「会議を更新」

当日の会議がカードとして取り込まれ、キャパシティバーに反映されます。
繰り返しの定例も正しく展開されます。

### 会議を評価する

取り込み直後は全部 `contribute` / 代替不可 の扱いです。**カードのメニューから自分の役割を設定してください。**

| 役割 | 意味 | 重要度 |
|---|---|---|
| decision | 自分が決める人 | 3 |
| contribute | 発言・貢献が要る | 2 |
| listen | 聞いているだけ | 1 |

**コスト = 会議時間（分）× 参加人数 ÷ 60**（人時）で計算されます。1時間×6人 = 6人時。

判定結果：

| 表示 | 条件 | 打ち手 |
|---|---|---|
| 辞退・委任候補 | `listen` かつ「代替可能」 | 出るのをやめる、誰かに任せる |
| 短縮・少人数化の交渉候補 | 高重要 かつ 高コスト | 30分に縮める、人数を減らす |

**「代替可能」を切らない限り辞退候補にはなりません。** 聞くだけでも自分が出る必要がある会議は、
代替可能をオフのままにしておけば候補から外れます。

---

## 9. 委譲の追跡

`type: delegated` のタスクを Waiting に置くと、次の3つを見張ります。

- `expected_by` を過ぎた
- 委譲から設定日数（既定7日）動きがない
- （Backlogに残っている場合）滞留

### 催促する

カードのメニュー → 「委譲メッセージを作成」で、依頼内容・期待返答日を含む定型文が生成されます。
編集してクリップボードにコピーし、Slack等に貼ってください。

**Vaultをチームで共有する機能はありません。** これは片方向の管理です。
価値は「こちらが忘れないこと」と「コストが見えること」にあります。

---

## 10. デイリープランニング

朝の並べ直しの儀式です。設定した時刻以降、その日はじめてボードを開いたときに提案されます。
手動で開くには `Ctrl+P` → 「デイリープランニングを開く」。

1. Ready のタスクが一覧で出ます
2. 今日やるものをオンにします
3. 上部に「選択 180分 / 使える時間 210分」と出るので、**収まる範囲で選びます**
4. 「今日の枠を確定」

確定すると、選んだタスクが **Ready の上位に並び替わります**。状態は変わりません（Doingへは自分で引きます）。

着手中（Doing）のタスクの見積も合計に含まれます。すでに持っている分を勘定に入れるためです。

**スキップも記録されます。** 実施率を見るためなので、やらなかった日も正直に閉じて構いません。

---

## 11. 週次サマリー

週のはじめにボードを開くと、3行だけ出ます。

```
滞留カード 4件
Expedite 6回
WIP超過 2回
```

数字を見るだけの機能です。増えているなら、上限や可処分時間の設定が実態と合っていない可能性があります。

## 12. 設定項目

| 項目 | 既定 | 説明 |
|---|---|---|
| WIP上限 | 3 | Doingに置ける件数 |
| 1日の可処分時間 | 360分 | 会議を引く前の作業時間 |
| フォローアップコスト | 10分 | Waiting 1件あたりの見えないコスト |
| 役割係数 decision / contribute / listen | 3 / 2 / 1 | 会議の重要度 |
| 高重要とみなす下限 | 3 | 交渉候補の判定に使う |
| 高コストとみなす下限 | 4人時 | 交渉候補の判定に使う |
| ICS購読URL | なし | 1行に1つ |
| 自動取得間隔 | 60分 | 0で自動取得しない |
| プランニング提案時刻 | 09:00 | この時刻以降の初回表示で提案 |
| 滞留とみなす日数 | 14日 | Backlogの滞留バッジ |
| 委譲の無変化を警告する日数 | 7日 | 委譲の無変化バッジ |
| タスクノートの作成先 | Tasks | 昇格・新規作成の保存先 |

## 13. frontmatter リファレンス

ボードを使わず直接編集しても構いません。編集は即座にボードへ反映されます。

```yaml
type: task          # task | meeting | delegated
status: doing       # backlog | ready | doing | waiting | done
sort_key: 1.5       # 列内の並び順（自動）
estimate: 60        # 見積（分）
due: 2026-08-01     # 期限（任意）
project: 基幹刷新    # 任意
ticket_url: https://...  # 任意。内容は複製しない
expedite: false     # 割り込みフラグ

# type: meeting のとき
meeting_role: contribute   # decision | contribute | listen
replaceable: false         # 代替可能か
attendees: 6               # 参加人数
meeting_start: 2026-07-19T10:00:00.000Z
meeting_duration: 60       # 分

# type: delegated のとき
assignee: 田中
delegated_at: 2026-07-19
expected_by: 2026-08-01
```

`status_changed_at` と `created_at` は自動管理です。**手で消さないでください。**
滞留・無変化の検知がこれを見ています。

## 14. 困ったとき

**「このノートは外部で編集されました」と出る**
別のデバイスやエディタで同じノートが変更されています。上書きを避けて読み直した状態です。
内容を確認してから操作し直してください。データは失われていません。

**会議が取り込まれない**
ICS URLが「公開」リンクか確認してください。非公開のカレンダーURLでは取得できません。
複数のカレンダーを登録している場合、片方が失敗しても他は取り込まれます。

**カードがボードに出てこない**
frontmatterに `type:` が無いか、`task` / `meeting` / `delegated` 以外になっています。

**同じ会議が二重に出る**
他のカレンダー同期プラグインと同じICSを購読していないか確認してください。

**Doingに移動できない**
WIP上限に達しています。何かを終わらせるか Waiting に逃がしてください。
どうしても必要なら、Expedite にすれば枠外で着手できます。

---

## 15. 現時点でできないこと

- チケットシステム（Backlog/Jira）とのAPI連携
- カレンダーへの書き戻し（読み取り専用）
- Vault共有によるチームでのタスク同期
- モバイルでのドラッグ&ドロップ
- 委譲タスクを作る専用コマンド（frontmatterを手で編集）
- リードタイム等のフロー指標、週次レポートの自動生成
